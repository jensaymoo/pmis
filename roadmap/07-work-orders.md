# Фаза 7 — Наряды и факт

> Диспетчер создаёт наряды, добавляет работы, закрывает с фактом и ресурсным фактом.

## Что делаем

- `components/orders/WorkOrdersGrid.vue` — `n-data-table`, пагинация, статус, даты, closed_at
- `components/orders/WorkOrderForm.vue` — модалка/форма:
  - **Создание:** title, picker листовых работ (plan_qty > 0, percent_done < 100); для секционированных — выбор участка; добавление нескольких work_order_task
  - **Закрытие:** по каждой строке — fact_qty + ресурсный факт; submit → `rpc('close_work_order', ...)`
- `components/orders/WorkOrderResourceFact.vue` — ввод ресурсного факта:
  - Конкретный ресурс: имя read-only + qty
  - Группа: мультиселект членов группы + qty каждому
  - Кнопка «Просмотр ресурса» → `ResourceFormModal` в режиме `readonly`

## Готово когда

- Создание наряда с листовыми работами → ок
- Добавление составной работы → ошибка
- Добавление работы на 100% → ошибка
- Секционированная работа без выбора участка → ошибка
- Закрытие с корректным фактом → percent_done обновляется, actual_start/end
- Пропущенный ресурсный факт → ошибка
- Ресурс не из группы → ошибка
- fact_qty = 0 → допустимо
- Закрытый наряд → read-only
- **Один ресурс в нескольких открытых нарядах** → факт суммируется при каждом закрытии
- Роллап percent_done по дереву работает
- При выдаче наряда диспетчеру показывается «план на сегодня» (дневной объём на текущую дату) как информационная подсказка

**Оценка:** 8–10 дней

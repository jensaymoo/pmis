## Что делаем

- `assets/bryntum.css` — стили Bryntum
- `pages/GanttPage.vue` — обёртка без padding
- `adapters/bryntumPostgrest.js`:
  - `loadProject()` — загрузка project + task, построение дерева, `loadInlineData`
  - `syncProject()` — маппинг полей (startDate↔start_date), сохранение parent_id при indent/outdent, обработка phantom ID
- `components/gantt/GanttView.vue`:
  - `<BryntumGantt>` с `<script setup>`
  - Русская локаль
  - `taskEdit: false` (встроенный редактор выключен)
  - Auto-save: `project.on('change')` → debounce 500мс → `syncProject()`
- **Кастомная модалка работы** (Naive UI `n-modal`, открывается по double-click): единая прокручиваемая форма с разделами (не переключаемые вкладки — все разделы видны одновременно, согласно `planning-task-editor.md` §2)
  - **Раздел «Атрибуты и календарный план»**: name, dates, duration, plan_qty, qty_unit
  - **Раздел «Участки»** (для листовых работ):
    - **Read-only таблица** `task_section`: наименование, план. объём, факт. объём, прогресс (%), факт. начало, факт. завершение
    - Создание/редактирование участков и точек — через модалку `SectionsGrid` (фаза 5, `components/references/SectionsGrid.vue`), вызываемую из раздела; реализация не дублируется
    - plan_qty по каждому участку
    - Поле plan_qty работы → read-only (= сумма участков)
  - **Раздел «Ресурсы»** (для листовых работ):
    - Список *_resource_plan назначений
    - Добавление: выбор вида → ресурс или группа → количество
  - **Раздел «Зависимости»** (для листовых работ): FS/SS/FF/SF + lag, согласно `planning-dependencies.md`
  - **Редактор дневного плана** (модалка второго уровня, открывается из ячейки календаря):
    - Плановый объём на день + валидация остатка
    - Назначения ресурсов (персонал, техника, материалы) — единая форма, без табов
    - Для секционированной работы — выбор участка (режим создания)

## Готово когда

- Логин планировщиком → Гант с WBS из сида
- Изменение имени/дат → auto-save → переживает F5
- Double-click → кастомная модалка Naive UI (НЕ попап Bryntum)
- Drag работы на другой уровень → parent_id сохранён
- Дочерние даты за рамками родителя → ошибка от сервера, уведомление
- Удаление работы с потомками → каскадное удаление
- Добавление участков → plan_qty = сумма участков, поле read-only
- **Таблица участков показывает факт/прогресс** (данные из закрытых нарядов)
- Участок с недостаточным числом точек → ошибка
- Ресурсные назначения сохраняются
- Календарная сетка для листовых работ; для составных и вех — недоступна с пояснением
- Редактор дневного плана открывается из ячейки календаря, а не из раздела формы работы
- Диспетчер не попадает на `/gantt`

**Оценка:** 10–14 дней
